# RB-API-002 - API Latency / Timeout Incident

## Overview

This runbook provides standardised response procedures for incidents
involving degraded responsiveness, high latency, or timeouts in AI-related
APIs and services.

It supports system availability and operational resilience.

---

## Purpose

This runbook enables teams to:

- Diagnose performance bottlenecks
- Restore acceptable response times
- Manage provider and infrastructure risks
- Maintain service reliability

---

## Applicable Systems

This runbook applies to:

- LLM inference APIs
- Embedding services
- RAG retrieval pipelines
- Vector database queries

---

## When to Use

Use this runbook when:

- Response times increase significantly
- Timeout rates rise
- Users report slow or stalled services
- Provider reliability issues are detected
